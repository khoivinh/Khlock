### Vite `.env` Fix
- Added `envDir: path.resolve(import.meta.dirname)` to `vite.config.ts` — Vite's `root` is set to `client/` so it was looking for `.env` in `client/` instead of the project root. This was why the Clerk publishable key wasn't being loaded and the login button was grayed out.

### CORS Fix
- Added `http://localhost:5174` and `http://localhost:5175` to `ALLOWED_ORIGINS` in `api/wrangler.jsonc` — Vite sometimes picks alternate ports when 5173 is in use

### Node Version Management
- Installed Volta and pinned Node 22.22.2 as default — Wrangler 4.77's bin script silently fails on Node 24 (child process spawn with IPC channel swallows all stdout/stderr)
- Added Volta PATH to end of `~/.zshrc` to override `/usr/local/bin/node` (v24)
- **Workaround for `npx wrangler` in terminal**: Run wrangler directly with `node --no-warnings --experimental-vm-modules ./node_modules/wrangler/bin/wrangler.js <command>` to bypass the broken bin script spawn

### Clerk Dashboard Config

- Confirmed app name is "Khlock" (was already correct despite devlog noting a typo)
- Allowed origins: not configurable in dev mode (dev instances allow all origins automatically)

## Decisions Made
- **API token over OAuth**: Wrangler's OAuth login doesn't work in non-interactive environments (Claude Code's Bash tool). Using `CLOUDFLARE_API_TOKEN` env var is more reliable and works everywhere.
- **Node 22 via Volta**: Node 24 causes silent wrangler failures. Volta chosen over nvm for simplicity.
